
### 1. 对比
这些概念都和用户身份管理息息相关。当哈基米登录某个网站时：
- 首先是身份标识（Identification）—— 哈基米告诉系统“我是哈基米”；
- 接着是身份验证（Authentication）—— 系统核实哈基米是否真的是哈基米本人；
- 最后是授权（Authorization）—— 系统授予哈基米访问某些资源的权限。
这些技术就是针对这个问题的解决方案。

| 技术                      | 定义                       | 工作原理                                                                  | 优点              | 缺点                                  | 应用场景                     | 常见问题                             |
| ----------------------- | ------------------------ | --------------------------------------------------------------------- | --------------- | ----------------------------------- | -------------------------- | --------------------------------- |
| **WWW-Authenticate**    | HTTP 基本认证方式              | 服务器返回 `WWW-Authenticate` 响应头，浏览器弹窗要求输入用户名密码，后续请求用 `Authorization` 头携带 | 简单，直接用浏览器支持     | 无法控制会话生命周期，用户体验差，不安全（明文或 Base64 传输） | 内网测试、临时认证                  | 为什么现在很少用？如何防止明文密码泄露？              |
| **Session-Cookie**      | 基于服务器会话和浏览器 Cookie 的身份管理 | 服务器保存 Session 数据，浏览器保存 Session ID（Cookie 中），请求时自动带上 Cookie            | 生命周期可控，支持存储复杂数据 | 仅适合浏览器，移动端兼容性差，需要服务器存储会话数据          | 网站登录状态维持                   | Session 如何实现分布式共享？Cookie 如何防止被窃取？ |
| **Token**               | 解决 Cookie 在非浏览器环境的兼容性问题       | 客户端保存 Token，每次请求手动在 HTTP 头中携带，服务器验证有效性                                | 跨平台，适合移动端和前后端分离 | Token 需要加解密，增加性能开销，易被窃取             | 移动端 API 鉴权、前后端分离应用         | Token 一旦泄露怎么办？如何设置过期和刷新？          |
| **JWT（JSON Web Token）** | 标准化的 Token 格式，内含签名       | Token 由三部分组成（Header.Payload.Signature），签名保证数据未被篡改，服务器只需验证签名           | 无需存储会话（无状态），跨平台 | Token 一旦泄露不可撤销；体积大，传输有额外开销          | 单点登录、API 认证、微服务通信          | JWT 如何防止重放攻击？如何缩小泄露风险？            |
| **SSO（单点登录）**           | 一次登录，多系统共享会话             | 通过 CAS（Central Authentication Service）集中验证用户身份，颁发跨系统可用的票据             | 提升用户体验，统一身份管理   | 系统间需要共享认证中心；实现复杂                    | 企业内部系统、跨业务系统整合             | 如何在不同域名下共享登录状态？如何防止 CSRF？         |
| **OAuth 2.0**           | 一种授权协议                   | 用户授权第三方应用访问自己在另一个网站上的资源，常用四种授权模式（授权码、简化模式、密码模式、客户端模式）                 | 不暴露密码，安全授权      | 协议复杂，需要正确实现安全流程                     | 第三方登录（微信、GitHub、Google 登录） | 授权码模式流程？如何防止授权码被劫持？               |
